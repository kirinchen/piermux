# Issue

Concrete problems, bugs, or requests. Each Issue answers **what** — a discrete piece of work with clear acceptance criteria.

## When to create an Issue

- A bug with a reproducer
- A feature request small enough that the user can state clear acceptance criteria
- A technical task (refactor, migration, tooling) with a concrete outcome

Not every Issue belongs to an Epic — standalone bugs and small enhancements are fine with just Sprint membership.

## Lifecycle

```
open → in_progress → resolved → closed
```

- `open` — logged but nobody started yet
- `in_progress` — an agent/human is actively working
- `resolved` — work done, waiting for verification
- `closed` — verified, archived
