# Issue

Concrete problems, bugs, or requests. Each Issue answers **what** вҖ” a discrete piece of work with clear acceptance criteria.

## When to create an Issue

- A bug with a reproducer
- A feature request small enough that the user can state clear acceptance criteria
- A technical task (refactor, migration, tooling) with a concrete outcome

Not every Issue belongs to an Epic вҖ” standalone bugs and small enhancements are fine with just Sprint membership.

## Lifecycle

```
open вҶ’ in_progress вҶ’ resolved вҶ’ closed
```

- `open` вҖ” logged but nobody started yet
- `in_progress` вҖ” an agent/human is actively working
- `resolved` вҖ” work done, waiting for verification
- `closed` вҖ” verified, archived
