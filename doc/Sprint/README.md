# Sprint

Time windows. Each Sprint file covers one iteration with a goal, committed Issues, daily notes, and a retrospective.

## Lifecycle

```
planning → active → review → done
```

- `planning` — Sprint proposed but not yet started
- `active` — currently running; **exactly one** Sprint should have this status
- `review` — Sprint ended; retrospective being drafted
- `done` — retrospective complete, archived

The SessionStart hook highlights whichever Sprint has `status: active`.

## Sprint ID convention

`SPRINT-<YYYY>-W<WW>` using ISO-8601 week date. Example: `SPRINT-2026-W17` covers the Monday of ISO week 17 in 2026 through the following Sunday.
