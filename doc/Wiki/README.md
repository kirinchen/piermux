# Wiki

Background knowledge, guides, and architecture decision records (ADRs).

Structure:

```
Wiki/
├── architecture-decisions/   — one ADR per decision
└── guides/                   — how-to, setup, onboarding
```

## Wiki vs ADR

- **Wiki page** — reference / how-to / evergreen knowledge. Updated as the system changes.
- **ADR** — a single decision, frozen once accepted. Superseded by new ADR when reversed.

Rule of thumb: if someone asks *"how do I X?"* → Wiki. If they ask *"why did we choose X?"* → ADR.
