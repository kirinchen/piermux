# Epic

Long-lived goals. Each Epic answers **why** — a direction that spans multiple Issues and possibly multiple Sprints.

## When to create an Epic

- A feature-set that will take 2+ weeks of sustained work
- A theme that ties several otherwise-independent Issues together
- A strategic direction that the team should be able to reference

Don't create an Epic for:
- A single bug fix → Issue
- A refactor that's purely mechanical → Issue (or just a Task)
- Something smaller than a week of work → Issue or Task

## Lifecycle

```
planning → active → done
              └──── cancelled  (when direction is abandoned)
```

Only one of {done, cancelled} is terminal.
